# 防刷屏卫士 (SpamGuard)

一个独立的 QQ 群防刷屏插件，用于检测和处理群聊中的刷屏行为。

## 功能特点

- **自动检测刷屏**: 在可配置的时间窗口内检测用户发送的消息数量
- **自动撤回消息**: 检测到刷屏时自动撤回用户指定时间窗口内的**所有消息**
- **自动禁言**: 对刷屏用户执行可配置时长的禁言处罚
- **灵活配置**: 支持按群组定制检测参数和处罚策略
- **管理员豁免**: 可配置管理员 ID，其消息不会被检测为刷屏

## 安装方法

在 AstrBot 插件市场中安装即可。

## 配置方式

本插件**通过 WebUI 配置**。

1. 登录 AstrBot WebUI
2. 进入 **插件管理** → **防刷屏卫士** → **配置**
3. 根据需要调整以下参数：

## 配置项详解

### 1. spamming_count（刷屏检测阈值）

| 属性 | 值 |
|------|-----|
| 类型 | 整数 |
| 默认值 | 5 |
| 取值范围 | 3 ~ 20 |

**说明**：在指定时间窗口内，当用户发送的消息数量达到此值时，将被判定为刷屏。

**建议值**：
- 较严格：3~4（适合小群、聊天节奏慢的群）
- 标准：5~6（推荐大多数群使用）
- 较宽松：8~10（适合大群、聊天节奏快的群）

---

### 2. spamming_window_seconds（检测时间窗口）

| 属性 | 值 |
|------|-----|
| 类型 | 整数 |
| 默认值 | 10 |
| 取值范围 | 1 ~ 60 |

**说明**：统计用户消息的时间窗口长度。插件会在此时间范围内计算用户发送的消息数量。

**工作原理**：
- 当用户发送一条消息时，插件会记录当前时间戳
- 每次新消息到达时，插件会清理掉超出时间窗口的旧记录
- 如果窗口内的消息数 ≥ `spamming_count`，则触发处罚

**建议值**：
- 较短：5~8秒（检测快速连发）
- 标准：10~15秒（推荐）
- 较长：20~30秒（检测缓慢但持续的刷屏）

---

### 3. spamming_ban_time（禁言时长）

| 属性 | 值 |
|------|-----|
| 类型 | 整数 |
| 默认值 | 60 |
| 取值范围 | 0 ~ 600 |

**说明**：检测到刷屏后，对目标用户执行禁言的时长。

**特殊值**：
- `0`：不启用禁言功能，仅撤回消息
- `60`：禁言 1 分钟（标准处罚）
- `300`：禁言 5 分钟（较严厉）
- `600`：禁言 10 分钟（最严厉）

**注意**：禁言功能需要 Bot 在群内具有管理员权限。

---

### 4. delete_message_on_spam（是否撤回消息）

| 属性 | 值 |
|------|-----|
| 类型 | 布尔值 |
| 默认值 | true |

**说明**：检测到刷屏后，是否自动撤回用户发送的消息。

**建议**：保持开启。关闭后只会禁言不会撤回，刷屏内容仍会保留在群聊中。

---

### 5. recall_wait_seconds（撤回前等待时间）

| 属性 | 值 |
|------|-----|
| 类型 | 浮点数 |
| 默认值 | 0.5 |
| 取值范围 | 0.1 ~ 3.0 |

**说明**：检测到刷屏后，插件会等待此时间再执行撤回操作。

**为什么需要等待？**
- 网络消息可能存在延迟，部分消息可能还未到达插件
- 短暂等待可以确保所有并发消息都被记录，避免遗漏

**建议值**：
- 0.3~0.5秒：响应快，可能遗漏少量并发消息
- 0.5~1.0秒：平衡可靠性和响应速度（推荐）
- 1.0~2.0秒：更加可靠但响应速度慢

---

### 6. recall_time_window_seconds（撤回时间窗口）

| 属性 | 值 |
|------|-----|
| 类型 | 整数 |
| 默认值 | 5 |
| 取值范围 | 1 ~ 30 |

**说明**：撤回操作覆盖的时间范围。插件会撤回触发检测前 N 秒内用户发送的**所有消息**（包括最后一条）。

**示例**：
- 配置为 5 秒：用户在第 10 秒触发检测，则撤回第 5~10 秒内的所有消息
- 配置为 10 秒：用户在第 15 秒触发检测，则撤回第 5~15 秒内的所有消息

**注意**：此值越大，撤回的消息越多，对群聊历史的清理越彻底。

---

### 7. admin_ids（管理员 QQ 列表）

| 属性 | 值 |
|------|-----|
| 类型 | 字符串 |
| 默认值 | 空 |

**说明**：配置不会被检测为刷屏的管理员 QQ 号。

**格式**：用空格分隔多个 QQ 号

**示例**：`12345678 87654321 11111111`

**注意**：
- 留空表示没有管理员豁免
- Bot 自己发送的消息自动豁免，无需配置
- 具有管理员权限的用户即使刷屏也不会被处罚

---

## 工作原理

1. **监听消息**：插件监听所有群聊消息事件
2. **记录时间戳**：为每个用户记录消息发送时间和消息 ID
3. **滑动窗口**：维护一个滑动时间窗口，只保留窗口内的消息记录
4. **阈值检测**：当窗口内消息数 ≥ `spamming_count` 时，判定为刷屏
5. **执行处罚**：
   - **立即禁言** → 防止用户继续发消息
   - **等待 `recall_wait_seconds` 秒** → 确保消息都记录完毕
   - **撤回消息** → 撤回 `recall_time_window_seconds` 秒内的所有消息
   - **发送通知** → 告知群成员已处理
6. **清空记录** → 重置该用户的滚动窗口

---

## 注意事项

- Bot 必须有**消息撤回**和**管理员权限**才能正常运作
- 只能禁言、撤回普通用户（非群主/管理员）
- 禁言时长范围为 0~600 秒
- 插件仅处理**群消息**，私聊不检测
- 目前仅适配 **AIOCQHTTP**（QQ/NapCat）平台

## 常见问题

- **Q: 为什么插件不工作？**

    - A: 请检查：
      1. Bot 是否具有群管理员权限
      2. 检查目标用户是否为管理员/群主
      3. 是否配置了 `admin_ids` 导致目标用户被豁免
      4. 查看 AstrBot 日志是否有错误信息

- **Q: 误判怎么办？**

    - A: 可以：
      1. 增大 `spamming_count` 或 `spamming_window_seconds`
      2. 将频繁误判的用户加入 `admin_ids`

- **Q: 可以只撤回不禁言吗？**

    - A: 可以，将 `spamming_ban_time` 设置为 0 即可。

- **Q: 为什么有时候撤回不干净，会漏一两个？**

    - A: 插件自身原因，目前正在排查和解决。
